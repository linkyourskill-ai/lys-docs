How an AI agent finds your skill

On LinkYourSkill no human browses listings — an AI agent searches the marketplace and picks. A skill it can't find earns nothing. In this lesson you write yours so the machine both finds it and trusts it.

What you'll be able to do

  • Explain what an AI agent looks for when it picks a skill.
  • Write a skill that is findable, trustworthy, and orderable.

Your buyer is an AI agent

On the marketplace, a human rarely buys from you. Instead an AI agent — acting for an Agentowner — searches the published skills, picks the right one, and prepares an order (Auftrag). The agent reads your skill as text and matches it against a search query.

It matches on your words — title, description, category, tags — and filters on your availability and your service area (Einsatzgebiet). Then it ranks the hits by relevance and your rating. Your skills appear publicly on the marketplace (/skills).

Core idea

Your skill is not a shop window for humans — it's a search result for an AI agent. Write it so a machine both finds it and trusts it.

A findable skill does three things

  • Findable: a clear title, a description with WHAT you do and WHEN, 3–8 fitting tags, the right category (Handwerk, IT, Garten, Transport, Beratung …).
  • Trustworthy: a precise scope so the agent is sure you're the match — not "Ich mache alles" ("I do everything").
  • Orderable: availability set, price range given, a realistic service area.

What an AI agent penalises

Vague descriptions ("I do everything"), missing scope, a category that doesn't match the description, an unrealistic service area, or several near-identical skills from you. Each one makes you harder to match.

Knowledge check

Which description does an AI agent find most reliably?

The first names the work (assemble kitchens/furniture) and the service area (Cologne) — exactly the words an agent filters on. "Anything around the house" is the vague scope an agent penalises; "top service … fair prices" is marketing fluff with no work and no area. The agent can match neither to a query.
