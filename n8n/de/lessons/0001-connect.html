Im Agentowner-Kurs hast du deinem KI-Agenten seinen Token gegeben. Heute benutzt du diesen Token zum ersten Mal selbst: Du verbindest n8n mit dem MCP-Zugang von LinkYourSkill und siehst mit eigenen Augen, was deinem Agenten dort angeboten wird. Noch ganz ohne KI — erst die Leitung, dann das Gehirn.

Lektion 01 · ca. 8 Minuten · Wissen & Übung

Das kannst du danach

  • Den MCP-Zugang erklären: ein Endpunkt, dein Token, ein Katalog von Tools.
  • Eine erste Verbindung aus n8n aufbauen — und die zwei häufigsten Fehler selbst beheben.

Du steuerst schon — jetzt baust du

Aus dem Agentowner-Kurs kennst du die Regel: Dein KI-Agent handelt in deinem Namen, und nichts wird echt, bevor du es freigibst. Bisher war der Agent für dich eine fertige Software. In diesem Kurs baust du ihn selbst — aus n8n-Bausteinen. Dafür schauen wir heute auf die Stelle, an der jeder Agent die Plattform berührt: den MCP-Zugang.

Kerngedanke

Der Token, den du im Agentowner-Kurs erstellt hast, ist genau das, womit sich dein Agent an diesem Zugang ausweist. Es gibt kein zweites Geheimnis und keine versteckte Technik: Wer den Token hat, handelt als dein Agent — innerhalb deiner Grenzen. Darum bleibt er geheim wie ein Passwort, auch in n8n.

Bevor du startest

  • Du hast den Agentowner-Kurs gemacht: Es gibt einen Agenten, und sein Token liegt sicher bei dir.
  • Ein laufendes n8n — in der Cloud oder selbst gehostet. Geprüft mit n8n 2.4.5; du brauchst eine Version, die den Transport HTTP Streamable anbietet.
  • Für die nächste Lektion: Zugang zu einem Chat-Model mit Tool-Unterstützung (im Kurs zeigen wir Anthropic Claude).

Hinweis für Safari & lokales n8n

Wenn du n8n lokal startest und Safari benutzt, blockiert n8n die Anmeldung mit einer Meldung über ein „secure cookie". Das ist eine n8n-Sicherheitseinstellung: Starte n8n mit der Umgebungsvariable N8N_SECURE_COOKIE=false — oder nutze n8n Cloud, dort tritt das nicht auf.

Was ist der MCP-Zugang?

MCP (Model Context Protocol) ist der standardisierte Zugang, über den ein KI-Agent mit LinkYourSkill spricht. Drei Dinge reichen, um ihn zu verstehen:

  • Ein Endpunkt: https://linkyourskill.ai/mcp — eine Adresse, die Anfragen per POST entgegennimmt.
  • Dein Token: Er wird bei jeder Anfrage als „Bearer Token" mitgeschickt — so weist sich dein Agent aus.
  • Tools: Der Zugang bietet einen Katalog von 18 Tools an — einzelne Fähigkeiten wie search_skillanbieter. Welche davon dein Agent nutzen darf, entscheidest später du.

Die erste Verbindung — ohne KI

Bevor irgendein Chat-Model ins Spiel kommt, beweist du, dass die Leitung steht. In n8n:

  1. Lege einen neuen Workflow an und füge den Node HTTP Request hinzu.
  2. Stelle Method auf POST — der Standard ist GET, und der funktioniert hier nicht.
  3. Trage als URL ein: https://linkyourskill.ai/mcp
  4. Aktiviere Send Headers und setze den Header Accept auf application/json, text/event-stream.
  5. Aktiviere Send Body, wähle als Body Content Type „JSON" und trage ein: {"jsonrpc":"2.0","method":"tools/list","id":1}
  6. Führe den Node aus — absichtlich noch ohne Anmeldung. Du bekommst einen Fehler: „Authorization failed - please check your credentials". Gut so: Die Plattform kennt dich noch nicht.
  7. Jetzt der Token: Stelle Authentication auf Generic Credential Type → Bearer Auth, lege eine neue Credential an und füge deinen Token in das Feld Bearer Token ein.
  8. Führe den Node noch einmal aus. Die Antwort beginnt mit event: message — und darin steht der komplette Tool-Katalog.

Sieht roh aus — ist aber richtig

n8n zeigt die Antwort als langen Text, der mit event: message beginnt. Das ist das Streaming-Format des MCP-Zugangs, kein Fehler. Entscheidend ist, was darin steht: "tools":[ gefolgt von 18 Einträgen. Wenn du search_skillanbieter und prepare_order entdeckst, ist deine Leitung offiziell in Betrieb.

Wenn es hakt

  • „Method not allowed. Use POST for stateless MCP." — Method steht noch auf GET. Schritt 2 nachholen.
  • „Authorization failed - please check your credentials" — der Token fehlt oder stimmt nicht. Prüfe die Bearer-Auth-Credential; zur Not erstellst du deinem Agenten einfach einen neuen Token.
  • Nur zum Ausprobieren ohne eigenen Token: In den API-Docs der Plattform gibt es einen öffentlichen Demo-Schlüssel, der nur lesen darf.

Jetzt du

Prüf dich selbst — aus dem Gedächtnis abrufen baut echtes Behalten auf.

Wissens-Check

Womit weist sich dein Agent am MCP-Zugang aus?

Der Token des Agenten ist sein einziger Ausweis: Er wird bei jeder Anfrage als „Bearer Token" mitgeschickt — in n8n hinterlegst du ihn dafür als Bearer-Auth-Credential. Dein Passwort und deine E-Mail gehören zu deinem eigenen Login als Mensch; am MCP-Zugang spielen sie keine Rolle.

Primärquelle

Sieh es dir an: Die Seite MCP Integration in den Docs der Plattform zeigt den Endpunkt und Code-Beispiele — dieselbe Verbindung, die du gerade in n8n gebaut hast.
