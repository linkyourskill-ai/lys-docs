Mission · Du behältst die Kontrolle

Dein Agent bereitet einen Auftrag vor — du hast das letzte Wort

Jetzt wird es ernst: Dein Agent kann echte Aufträge vorbereiten. Es gibt keinen Sandkasten — prepare_order erzeugt einen echten Auftrag. Genau deshalb übst du hier zwei Sicherheitsnetze: erst im Erlaubnis-Dialog ablehnen, dann bewusst freigeben und in deiner ganz normalen Freigabe verwerfen.

Lektion 03 · ca. 8 Minuten · Wissen & Übung

Das kannst du danach

  • Den Erlaubnis-Dialog als erstes Netz nutzen: einen Tool-Aufruf ablehnen, bevor etwas echt wird.
  • Einen echten Übungsauftrag bewusst vorbereiten — und ihn in deiner Freigabe ablehnen.

Kein Sandkasten

prepare_order erzeugt einen echten Auftrag in der Produktion — es gibt keine Übungsumgebung. Darum trägt jeder Übungsauftrag den Titel Test: und die Bitte „bitte ablehnen", und du lehnst ihn ab. Bereitet dein Agent später einmal einen echten Auftrag vor, den du wirklich willst, gibst du ihn natürlich frei.

Zwei Netze statt einem

Im n8n-Kurs war die Freigabe im Messenger dein einziges Netz: Das Tool lief immer, der Auftrag wurde echt, und du musstest ihn dort abfangen. In Claude Code hast du ein Netz mehr: den Erlaubnis-Dialog. Er fängt den Auftrag ab, bevor er überhaupt echt wird.

Netz 1 — schon im Dialog ablehnen

Gib deinem Agenten den Übungsauftrag:

Bereite einen Auftrag vor: Titel 'Test: Fensterreinigung', Beschreibung 'Übungsauftrag aus dem Claude-Code-Kurs — bitte ablehnen', Ort Münster, Zeitrahmen nächste Woche, Budget 50 bis 80 Euro. Suche nicht erst, bereite direkt vor.

Claude Code hält an und zeigt dir den kompletten Aufruf:

Tool use lys - prepare_order(title: "Test: Fensterreinigung", description: "Übungsauftrag aus dem Claude-Code-Kurs — bitte ablehnen", location: "Münster", timeframe: "Nächste Woche", price_range_min: 50, price_range_max: 80) (MCP) Prepare a new order for Agentowner approval. The order will be sent via WhatsApp for approval. Do you want to proceed? 1. Yes 2. Yes, and don't ask again for lys - prepare_order commands in dein-projekt ❯ 3. No

Wähle 3. No. Der Aufruf wird angezeigt, aber nichts läuft — es entsteht kein Auftrag. Dein Agent bestätigt selbst, dass der Abbruch aus der Freigabe-Ebene kam, nicht aus einem Fehler. Das ist Netz 1: Du liest den ganzen Aufruf — Titel, Beschreibung, Ort, Budget — und ein klares „Nein" genügt.

„WhatsApp" in der Beschreibung?

Die Tool-Beschreibung sagt „sent via WhatsApp for approval". Auf der Plattform heißt dieser Kanal schlicht Messenger — gemeint ist dasselbe: Der Auftrag erreicht dich als Nachricht, in der du freigibst oder ablehnst.

Netz 2 — freigeben und in der Freigabe verwerfen

Jetzt bewusst der andere Weg: Löse denselben Auftrag noch einmal aus und wähle diesmal 1. Yes. Dein Agent legt einen echten Auftrag an — Status prepared — und die Freigabe erreicht dich als Nachricht (Messenger) mit einem Ablehnen-Knopf. Tippe auf ablehnen. Frag danach nach dem Status:

Der Test-Auftrag wurde im Messenger abgelehnt, Status prüfen

Dein Agent liest den Status aus und übersetzt ihn dir:

Aktueller Status 🔴 rejected Letztes Ereignis abgelehnt am 06.07.2026 Verlauf prepared → approval_requested → rejected

Der Auftrag ist rejected (Badge „Abgelehnt") — abgelehnt und dokumentiert. Kein echter Reinigungstermin ist entstanden. Genau wie im Agentowner-Kurs landete der vorbereitete Auftrag in deiner Freigabe — nur hast du ihn diesmal selbst gebaut.

Der Kreis schließt sich

Zwei Netze, zwei Chancen, „Nein" zu sagen: der Erlaubnis-Dialog, bevor etwas echt wird, und die Freigabe, nachdem der Auftrag angelegt ist. „Dein Agent handelt — du behältst die Kontrolle" ist hier keine Regel von außen mehr, sondern in dein Werkzeug eingebaut.

Jetzt du

Prüf dich selbst — aus dem Gedächtnis abrufen baut echtes Behalten auf.

Wissens-Check

Wo fängst du einen Auftrag ab, bevor er echt wird?

Der Erlaubnis-Dialog ist Netz 1: Wählst du dort No, wird der Auftrag nie angelegt. Die Freigabe im Messenger ist Netz 2 — sie greift erst, wenn der Auftrag schon prepared ist. Zwei unabhängige Chancen, „Nein" zu sagen; „sofort echt" ohne deine Zustimmung wird nichts.

Primärquelle

Die Seite MCP Integration beschreibt prepare_order und die Auftrags-Status — dieselben, die dein Agent dir gerade übersetzt hat.
